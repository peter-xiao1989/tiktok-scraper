# 多维表仪表盘搭建方案(海外游戏发行)

> 数据底座(YB8TbS「TT经营数据中心」)已由脚本自动同步,本文是仪表盘的设计 +
> 在飞书 UI 的搭建步骤。**飞书开放 API 不支持程序化创建图表组件,图表必须在 UI
> 手动配置**,所以这里把每张图设计到"照着点就能搭"。

## 数据底座(已自动同步,无需手动维护)

| 多维表 | 内容 | 刷新 |
|---|---|---|
| **经营概览(每日)** | 每日总消耗/收入/当日ROI/累计ROI/环比/新增/消耗最高/**风险提示**/**经营概要** | 每次导入 |
| 项目维度经营表 | 日×项目组:消耗/收入/累计ROI/新增 | 每次导入 |
| 各产品经营日报表 | 日×游戏:消耗/ROI/新增/活跃/启动成功率/eCPM/点击率… | 每次导入 |
| 投放日报-产品维度 | 日×游戏:消耗/ROAS/活跃度/人均广告次数 | 每次导入 |
| 投放日报-素材维度 | 日×素材:消耗/ROAS/展示/CTR/CPM | 每次导入 |
| 分时素材效果表 | 当日分时:素材级消耗/ROAS | 每小时 |

> 所有表已去掉"序号/类别"列;数值/百分比/日期列类型正确(图表能按数轴/日期轴聚合)。

## 业务口径(看数据前必须懂,否则会误判)

- **当日ROI**(=当日收入/当日消耗):游戏发行**首日天然偏低**(0.3~0.8),靠后续 LTV 回收,**不能拿它<1 当亏损**。
- **累计ROI**(=累计收入/累计消耗):**真正的盈利/回本指标**。<1 未回本,目标随天数爬向 1.0+。
- **最新一天产品收入 T+1 结算**(16点后到):当天显示收入0/ROI0 是"待结算",不是亏损。
- 起量初期消耗从极小基数翻几倍是正常起量,非异常。

## 仪表盘一:经营总览(管理层一眼看全局)

**数据源:经营概览(每日)**

| 组件 | 类型 | 配置 | 看什么 |
|---|---|---|---|
| 今日消耗 | 指标卡 | 取最新日「总消耗」 | 当日投放规模 |
| 今日新增 | 指标卡 | 最新日「新增用户」 | 拉新规模 |
| 累计ROI | 指标卡 | 最新日「累计ROI」,目标线1.0 | **回本进度** |
| 在投项目数 | 指标卡 | 最新日「在投项目数」 | 盘子宽度 |
| 消耗 & 新增趋势 | 折线图(双轴) | X=日期,Y1=总消耗,Y2=新增用户 | 投放节奏 vs 拉新 |
| 累计ROI趋势 | 折线图 | X=日期,Y=累计ROI,加 1.0 参考线 | 是否在爬向回本 |
| **风险看板** | 表格视图 | 列:日期/经营概要/风险提示,按日期倒序,**筛选 风险提示≠✅正常** | **一眼看异常天** |

> 「经营概要」「风险提示」是脚本已经算好的自然语言总结,团队不用自己算,直接读。

## UI 搭建步骤(通用)

1. 打开 base → 底部「+」→ 新建**仪表盘**
2. 仪表盘内「添加组件」→ 选图表类型(指标卡/折线/条形/饼图/表格)
3. **数据源**选对应多维表 → 配 X 轴(分类/日期)、Y 轴(数值/聚合方式)、分组、筛选
4. 折线/趋势图的"日期"轴直接选「日期/统计周期」字段(已是日期类型,能按天聚合)
5. "最新日"类组件:加筛选条件 日期 = 最大值,或用"最近1天"

## 后续可增强(数据补齐后)

当前数据没有**留存(次留/7留)、付费率、LTV曲线**——这几个是海外发行最核心的质量指标。若 TikTok/产品后台能取到,补进原表后,可加:留存漏斗、LTV回收曲线、付费率对比。届时仪表盘价值再上一个台阶。
