# 衍生表静态值化改造

## 为什么

6 张衍生表全是公式,实时依赖投放/产品原表(几万个 SUMIFS/SUMPRODUCT)。飞书重算不智能:**源表动一行就大批重算所有依赖公式** → 文档级阻塞("一动就崩")。2026-06-11 清理投放原表重复行时整表卡死,即此因。

## 方案

把衍生表从「飞书公式」改成「node 算好值 + 写纯静态值」。衍生表零公式 → 源表怎么动都不触发重算 → 永不阻塞。**实时性不变**:更新仍由每次导入后的脚本触发(频率一样),只是脚本直接写值、不写公式。

- 导入时:node 读源表 → 内存算(groupby+sum+排序) → 写静态值
- 非导入的改动(清理/删行/手改):衍生表是死值,完全不重算

## 注意

- 写静态值不触发公式重算,但大批写入仍有开销 → 分块(≤200行/批)+ 限流重试(feishuReq 已升级指数退避)
- 改造期间数据照常跑:每张表是「先有静态值版、验证、再切换」,不留空窗
- 排序逻辑搬到 node(LARGE/MATCH → JS sort),比飞书公式更直观可控
- 1-decimal/百分比/日期格式:写值时按列类型 round + 设 number format(沿用 classify/applyColumnFormats)
